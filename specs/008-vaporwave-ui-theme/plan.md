# Implementation Plan: Vaporwave Win95 UI Theme and Repository Credit

**Branch**: `008-vaporwave-ui-theme` | **Date**: 2026-05-24 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/008-vaporwave-ui-theme/spec.md`

## Summary

Transform the CYCLES game UI into a nostalgic 1990s desktop operating system aesthetic using a vibrant bisexual fuchsia orchid vaporwave color palette. Add a persistent Codeberg repository credit link. Add Reset and Undo game controls with visually distinct styling. Ensure the game is fully responsive (320px–1920px), installable as a Progressive Web App, WCAG 2.1 AA accessible (dark mode by default), and visually consistent across Firefox and Chromium browsers. All visual changes must be covered by automated visual regression tests.

## Phase 0: Research Summary

All clarifications resolved. See [research.md](research.md) for full details.

Key decisions:
1. **Visual regression**: `@vitest/browser` + Playwright snapshots
2. **PWA setup**: `vite-plugin-pwa` with Workbox
3. **Responsive strategy**: CSS custom properties + `vmin` + `@media` queries
4. **WCAG testing**: Axe-core in tests + Lighthouse + manual verification
5. **Typefaces**: VT323 (display) + Space Mono (body) via Google Fonts
6. **Color palette**: 12-token CSS custom property system with defined hex values

## Phase 1: Design Summary

### Data Model
See [data-model.md](data-model.md). Three primary entities:
- **Theme Configuration**: CSS custom properties for colors, fonts, shadows, borders
- **PWA Manifest**: App metadata for browser installation
- **Visual Regression Snapshot**: Reference images for automated comparison

### Contracts
See [contracts/ui-components.md](contracts/ui-components.md). Three component interfaces defined:
- **Modal**: Props, behavior (open/close triggers), styling (Win95 window chrome), accessibility (`<dialog>`, focus trapping)
- **MenuBar**: Props (callbacks), layout (responsive), repository link contract
- **Button**: Props (variant, disabled), styling states (default, hover, active, disabled). Includes `reset` and `undo` variants with distinct colors/icons for clear differentiation.
