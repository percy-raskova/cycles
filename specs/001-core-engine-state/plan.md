# Implementation Plan: Core Engine — State and Legality

**Branch**: `001-core-engine-state` | **Date**: 2026-05-20 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/001-core-engine-state/spec.md`

## Summary

Define an immutable `GameState` type and implement `legalPlacements(state)` and `legalJoins(state)` as pure functions over that state. The engine enforces all geometric and topological constraints of CYCLES (queen-lines, no coin blocking, no duplicate edges, no edge crossings) without any move-application or UI logic. Exit criteria are property-based tests (fast-check) proving that every generated move passes an independent hand-written oracle, and that the integer-coordinate segment-intersection planarity checker is robust against adversarial input.

**Scale/Scope**: Single 7×7 grid, 12 coins, 2 players — combinatorially bounded

## Phase 1: Design & Contracts

### Data Model

See `data-model.md` for full definitions.

**Key decision**: `GameState.coins` is a `ReadonlyMap<string, Coin>` keyed by `${row},${col}` for O(1) occupancy checks. `GameState.edges` is a `readonly Edge[]` because edge count is bounded (≤ C(12,2) = 66) and sequential scan is fast enough. `passCount` is added to GameState as specified in the sprint.

**New exports** (in addition to existing types):
- `legalPlacements(state): readonly Position[]`
- `legalJoins(state): readonly [Position, Position][]`

### Contracts

No external interfaces. The engine exposes a purely functional API consumed by `src/ui/`. The contract is implicit in the type signatures.
